Maze Game
You, a recent graduate from Mines have decided to make an amazing game that is procedurally generated from cubes. You have the code that generates the world and it is beautiful but lacking action so you decide to spruce it up with some A.I. that will choose an optimal path to get from point A to point B. Given the input file that lists all of the cubes in your world and the starting and ending positions, answer the following questions:
  • 1. What is the shortest number of moves needed to get from the starting point to the ending point?
  • 2. Print the valid path to a txt file in the same format as the input file

Movement Constraints
  • 1. Travel is restricted to along the top of a cube
  • 2. Diagonal movement is not allowed, only movement along the faces/edges of the cubes
  • 3. Vertical changes are allowed but only for deltas of one cube

File Information
  • 1. Each line of the file represents a cube that occupies a space in a three dimensional grid using x,y,z notation with the y axis pointing up.
  • 2. x,y,z >= 0
  • 3. The starting point will have an additional element (x,y,z,s)
  • 4. The ending point will have an additional element (x,y,z,e)
